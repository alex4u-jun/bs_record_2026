# 2026 기록실 — 배포 안내

## 폴더 구조
GitHub 저장소 `bs_record`에 아래처럼 올리면 됩니다.

```
bs_record/
├─ records.html          ← 새 기록실 (시즌 전환 지원)
├─ records.js            ← 기록실 로직
├─ merge.html            ← 경기 파일 정리 도구
├─ 2025/                 ← 작년 아카이브 (그대로 보존)
│   ├─ players.json
│   ├─ players_introduce.json
│   └─ gamerecord_1.json ~ gamerecord_8.json
└─ 2026/                 ← 새 시즌 (헌터스 vs 리더스)
    ├─ players.json              (빈 배열 [])
    ├─ players_introduce.json    (29명 명단)
    └─ gamerecord_1.json         (빈 경기)
```

## 업로드 순서
1. 저장소에 `2025/`, `2026/` 폴더를 만들고 각 데이터 파일을 올립니다.
   - 기존에 루트에 있던 작년 파일들(players.json, gamerecord_1~8.json 등)은 `2025/`로 옮기세요.
2. 루트에 `records.html`, `records.js`, `merge.html`을 올립니다.
3. 끝! `records.html`을 열면 기본으로 2026 시즌이 보이고,
   우측 상단 토글로 2025 아카이브를 오갈 수 있습니다.

## 시즌 동작
- **2026**: 아직 경기가 없어 "경기 기록이 아직 없습니다"가 표시됩니다.
  선수단 탭에는 29명(헌터스 15 / 리더스 14)이 나옵니다.
- **2025**: 작년 8경기 272타석 기록이 그대로 보존됩니다 (CSIA vs Leaders).

## 새 경기 기록하기
1. 기존 입력 페이지(index.html)로 경기를 기록하고 JSON을 내보냅니다.
2. `merge.html`에 그 파일(들)을 끌어다 놓으면 중복·빈 경기를 자동 정리해
   `gamerecord_merged.json` 하나로 받습니다.
3. 받은 파일을 `2026/gamerecord_1.json`(또는 _2, _3 …)으로 올립니다.
   기록실은 `2026/gamerecord_1~12.json`을 모두 읽습니다.
4. 선수 스탯 누적(`2026/players.json`)도 갱신해 올리면 순위에 반영됩니다.

## 선수 정보 채우기 (선택)
`2026/players_introduce.json`의 각 선수에 포지션·등번호가 비어있습니다(`-`).
나중에 정해지면 이 파일에서 `"position"`, `"backNumber"` 값을 채워 올리면
선수단·선수 상세에 표시됩니다. 주장은 `"captain": true`로 표시됩니다.

## 팀 이름이 또 바뀌면
`records.js` 상단의 `SEASONS` 설정에서 팀 식별자·표시이름·색상을 바꿀 수 있습니다.
2027 시즌을 추가하려면 `SEASONS`에 `"2027"` 블록을 추가하고 `2027/` 폴더를 만들면 됩니다.
